1. Tải file cấu hình OpenVPN

File .ovpn chứa chứng chỉ và cấu hình để client kết nối tới gateway. Mỗi user nên dùng tên riêng để dễ quản lý.

2. Cài client OpenVPN

Windows
  1. Tải OpenVPN Connect hoặc OpenVPN GUI.
  2. Mở app → import file .ovpn vừa tải.
  3. Bấm Connect.
macOS
  1. Cài Tunnelblick (miễn phí).
  2. Double-click file .ovpn → Tunnelblick sẽ import.
  3. Click icon Tunnelblick trên thanh menu → Connect.
Android
  1. Cài OpenVPN for Android (F-Droid) hoặc OpenVPN Connect (Play Store).
  2. Mở app → Import → chọn file .ovpn.
  3. Bấm dấu kết nối.
iOS
  1. Cài OpenVPN Connect trên App Store.
  2. Gửi file .ovpn qua AirDrop / iCloud / Mail rồi mở bằng OpenVPN.
  3. Bấm Add → Connect.
Linux (CLI) 
sudo apt install openvpn
sudo openvpn --config <file>.ovpn

3. Chống IPv6 leak (quan trọng)

File .ovpn đã có sẵn chỉ thị block-ipv6 (OpenVPN ≥ 2.5) tự động chặn IPv6 trên client khi VPN kết nối. Nếu kiểm tra ipleak.net vẫn thấy IPv6 lộ IP thật (Vietnam), hãy tắt IPv6 ở hệ điều hành:
Windows
  1. Settings → Network → Change adapter options.
  2. Click chuột phải vào card mạng (Wi-Fi / Ethernet) → Properties.
  3. Bỏ tick Internet Protocol Version 6 (TCP/IPv6) → OK.
macOS 
networksetup -setv6off Wi-Fi
networksetup -setv6off Ethernet
Linux 
sudo sysctl -w net.ipv6.conf.all.disable_ipv6=1
sudo sysctl -w net.ipv6.conf.default.disable_ipv6=1
# Vĩnh viễn: thêm 2 dòng trên vào /etc/sysctl.conf
Android / iOS Một số client OpenVPN (OpenVPN for Android, OpenVPN Connect) có tuỳ chọn IPv6 → Disable hoặc Block IPv6 trong cài đặt VPN profile — bật lên. Nếu không có thì đặt thiết bị ở chế độ chỉ dùng IPv4 (Settings → APN → IPv4 only đối với mạng di động).

Khi nào cần cài CA?

Chỉ cần nếu bạn bật MITM để soi traffic HTTPS. Nếu MITM tắt, traffic chạy thẳng qua proxy upstream và không cần cài gì thêm.
⬇ Tải CA (.pem) ⬇ Tải CA (.cer)

Hướng dẫn cài CA

Windows (toàn hệ thống)
  1. Tải file .cer ở trên.
  2. Double-click → Install Certificate → Local Machine → Next.
  3. Chọn Place all certificates in the following store → Browse → Trusted Root Certification Authorities → OK → Next → Finish.
  4. Có thể cần restart trình duyệt.
macOS (Keychain)
  1. Tải file .pem.
  2. Mở Keychain Access → File → Import Items → chọn file.
  3. Trong tab "System", tìm mitmproxy → double-click → Trust → "When using this certificate" → Always Trust.
  4. Đóng popup, nhập mật khẩu admin.
Android
Android 7+ chỉ tin user-CA cho app có cấu hình network-security-config. Nhiều app (Chrome, banking, ...) sẽ không tin user-CA — cần root để cài system-CA.
  1. Tải file .cer.
  2. Settings → Security → Encryption & credentials → Install a certificate → CA certificate → chọn file.
  3. Đặt tên (vd: "Proxy Checker MITM").
iOS / iPadOS
  1. Tải file .pem trên Safari → cho phép profile được tải.
  2. Settings → General → VPN & Device Management → Profile → tìm mitmproxy → Install.
  3. Settings → General → About → Certificate Trust Settings → bật toggle cho mitmproxy.
Firefox (mọi OS)
  1. about:preferences#privacy → cuộn xuống Certificates → View Certificates → Authorities → Import.
  2. Chọn file .pem → tích "Trust this CA to identify websites".

Bật/tắt giám sát traffic (MITM)

Khi bật MITM, traffic HTTPS sẽ được giải mã để xem payload. Client phải cài CA cert trước (xem tab CA), nếu không trình duyệt sẽ báo lỗi cert.
Traffic vẫn đi qua proxy upstream (cùng IP exit) như bình thường — chỉ thêm bước decrypt ở giữa.
